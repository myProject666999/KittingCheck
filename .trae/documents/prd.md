## 1. 产品概述

成品入库齐套检查系统，面向成品仓收货人员和生产计划人员，解决销售订单多料号、多批次入库时齐套状况难以实时掌握的问题。系统在成品入库环节即核对每个订单的齐套情况，缺件一目了然，齐套率达100%才放行发货，未齐套订单挂起并提醒生产补齐，杜绝发货时才发现缺件导致交付延误的风险。

- 目标用户：成品仓收货员、生产计划调度员
- 核心价值：入库即检、实时齐套、缺件可追溯、放行有依据

## 2. 核心功能

### 2.1 用户角色

| 角色 | 权限说明 |
|------|----------|
| 仓库收货员 | 扫码/手工录入入库记录，查看订单齐套状态 |
| 生产计划员 | 维护订单齐套清单，查看齐套看板，审批放行/挂起 |

### 2.2 功能模块

1. **订单管理**：销售订单的创建/导入，展开为应入库料号明细清单
2. **入库核对**：逐笔录入成品入库记录，实时累计已入数量，自动核算齐套率
3. **齐套看板**：按订单展示齐套进度，颜色区分齐套/缺件状态，支持筛选和搜索
4. **放行管理**：齐套率100%的订单自动标记可放行，未齐套订单挂起并生成缺件提醒

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 订单管理 | 订单列表 | 展示所有销售订单，支持按订单号/客户/状态筛选，显示齐套率概览 |
| 订单管理 | 新建/编辑订单 | 录入订单基本信息，添加应入库料号及数量明细 |
| 订单管理 | 订单详情 | 展示订单下所有料号的应入/已入/缺件数量明细 |
| 入库核对 | 入库录入 | 选择订单和料号，录入本次入库数量和批次号 |
| 入库核对 | 入库记录 | 查看历史入库流水，按订单/料号/批次筛选 |
| 齐套看板 | 进度看板 | 卡片式展示各订单齐套进度，颜色标识齐套/缺件/挂起状态 |
| 齐套看板 | 缺件汇总 | 汇总所有未齐套订单的缺件料号及差异数量 |
| 放行管理 | 放行列表 | 展示已齐套可放行和已挂起的订单，支持手动放行/挂起操作 |

## 3. 核心流程

用户创建销售订单并配置应入库料号明细 → 仓库收货员逐笔录入入库记录 → 系统实时累计已入数量并更新齐套率（写入Redis缓存）→ 齐套率达100%自动标记可放行 → 未齐套订单保持挂起，系统生成缺件提醒 → 生产计划员在看板监控齐套进度，确认补齐后放行发货。

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深蓝(#1E3A5F) + 翠绿(#10B981)作为齐套完成色 + 琥珀(#F59E0B)作为缺件警告色 + 红色(#EF4444)作为挂起色
- 按钮风格：圆角(8px)，主操作实心填充，次操作描边
- 字体：思源黑体/Noto Sans SC，标题18px加粗，正文14px常规
- 布局风格：左侧导航栏 + 顶部面包屑 + 内容区卡片式布局
- 图标风格：线性图标(lucide-react)

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 订单管理 | 订单列表 | 表格布局，状态标签彩色徽章，齐套率进度条，顶部筛选栏 |
| 订单管理 | 新建订单 | 分步表单，基本信息+料号明细表格，料号行可增删 |
| 订单管理 | 订单详情 | 上下布局：订单信息卡片 + 料号明细表格(每行显示应入/已入/差异) |
| 入库核对 | 入库录入 | 表单卡片：订单下拉选择→料号联动→数量+批次输入 |
| 入库核对 | 入库记录 | 时间线/表格布局，每条记录显示时间、料号、数量、操作人 |
| 齐套看板 | 进度看板 | 网格卡片布局，每卡片显示订单号+客户+环形齐套率图+状态标签 |
| 齐套看板 | 缺件汇总 | 表格布局，按料号汇总缺件，红色高亮差异数量 |
| 放行管理 | 放行列表 | 双栏布局：左侧可放行(绿色)、右侧已挂起(红色)，支持拖拽或按钮操作 |

### 4.3 响应式设计

- 桌面端优先(1920px/1440px/1280px)，表格和看板充分利用宽屏
- 平板端(768px-1024px)表格可横向滚动，看板卡片从4列变2列
- 移动端暂不做强制适配，以桌面和平板为主
